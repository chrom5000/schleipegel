# 3D-Hero Phase 3 (Seekartendaten + Animation) — Implementierungsplan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Seezeichen und Tiefenzahlen in „Revier 3D", die Windpartikel-Animation in der 3D-Szene, und glatte Ufer (hochaufgelöste Wassergeometrie).

**Architecture:** Seezeichen und Tiefenzahlen kommen als **gebackene GeoJSON-Dateien** (`seamarks.json`, `depths.json`) und werden als Quellen+Layer **direkt im STYLE_3D-Dokument** definiert — damit überleben sie `setStyle` ohne Re-Add-Logik und erscheinen nur in der 3D-Ansicht (Seekarte hat sie nativ). Die Windpartikel laufen auf einem Canvas-Overlay über der Karte: Partikel leben in Lon/Lat, `map.project()` bringt sie pro Frame auf den Schirm — funktioniert damit automatisch bei Pitch/Rotation/Zoom in beiden Ansichten. Die Wasserfläche wird neu aus OSM in voller Auflösung gebacken; die Terrain-Wassermaske rastert das Polygon per PIL statt Punkt-in-Polygon (schnell genug für viele Stützpunkte).

**Tech Stack:** Overpass API (Bake-Zeit), PIL-Polygon-Rasterung, MapLibre circle/symbol-Layer, Canvas 2D.

**Spec:** `.claude/specs/2026-07-12-3d-hero-design.md` · Phasen 1+2 fertig.
